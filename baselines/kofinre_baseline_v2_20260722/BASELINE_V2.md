## 결과 1 — D6 정의서 전수 (spec 카테고리, n=14,278)

전체 smelly 87.4% (v1 대비 +0.0%p 수준). **S7 미정의약어만 30.4% → 32.5% (+2.1%p)** —
약어 버그 수정 효과가 S7에 국소적으로 나타나고 나머지 9종은 완전 불변. spec 카테고리이므로
S1 재귀속은 미발동(1.7% 유지). 즉 버그 수정이 다른 스멜을 오염시키지 않음이 확인됨.

## 결과 2 — D5 공개 RFP (rfp 카테고리, n=10,899)

동일 데이터를 spec/rfp 모드로 각각 실행하여 S1 재귀속 효과를 대조:

| 스멜 | spec 모드 | rfp 모드 | Δ |
|---|---:|---:|---:|
| S1 복합 의무 | 84 (0.8%) | 31 (0.3%) | **−53** |
| S18 추적 ID 부재 | 2,929 | 2,983 | **+54** |
| S2·S5·S7·S8 | 불변 | 불변 | 0 |
| 전체 smelly | 85.8% | 85.8% | ~0 |

**해석**: RFP에서 S1로 잡히던 84건 중 53건이 실제로는 '1 ID 다세부' 묶음 구조였고,
이는 v2.9.3에서 S18(세부 식별자 부재)로 재귀속되었다. 전체 검출률은 불변 —
결함이 제거된 것이 아니라 올바른 유형으로 재분류됨. 잔존 S1 31건만이 묶음 마커 없는
단일 절 내 진성 복합 의무이다. 이는 §3.2의 "묶음 자체는 결함이 아니라 식별자 부재가
문제"라는 논거를 코드 수준에서 실증한다.
