# 실험 — D6: 실기업 요구사항 정의서 대규모 검증 (9개 금융 프로젝트)

> **D6 데이터셋 / KoFinRe v2.8 19종 평가**
> D4(실기업 4모듈 257건) 결론의 55배 규모 외적 타당성 검증.

---

## 데이터 출처

2011~2020년 수행된 한국 금융기관 9개 SI 프로젝트의 요구사항 정의서 xls/xlsx **140개 파일** (비공개 자료).
모듈당 최신 버전만 채택하여 **116개 파일 → 요구사항 행 14,278건** 추출.

**본 저장소에는 익명화된 통계만 포함한다** — 원문 문장·파일은 LOCAL_ONLY (D4와 동일한 윤리 프로토콜, 논문 §8.5c):

- 프로젝트 코드화: P-A ~ P-I (실기관-코드 매핑은 로컬 비공개 폴더에만 보관)
- 본문 마스킹: 기관명 873건 → `[ORG]`, 인명 188건 → `[PERSON]`, 벤더 50건 → `[VENDOR]`, URL 3건
- P-I 1개 파일은 DRM 암호화(SCDSA002 컨테이너)로 분석 제외

## 핵심 발견

1. **D4 결론의 대규모 재현** — S2 불완전 + S5 주체누락 지배 구조가 D4(91%/81%, 257건)에서
   D6(74.4%/59.0%, 14,278건)로 유지. "한국 SI 정의서 관행" 가설이 9개 기관·10년치에서 성립.
2. **원자성(S1) 우수 재확인** — 1행 1요구사항 양식의 구조적 강제 효과 (1.7%).
3. **MorphDetector ablation 재현** — kiwipiepy 활성 시 형태소 민감 smell(S4/S9/S5)만 증가
   (10종 기준 S4 +417, S9 +428), 논문 §8.5d의 D4 ablation(+55)과 동일 방향.
4. 프로젝트 편차: 74.5% ~ 96.6%. 최대 규모 P-B(2017)가 84.4%(10종 기준)로 상대적 양호.

## 재현 방법 (로컬 원본 보유 시)

```bash
# 1. 정의서 xls → 익명화 JSONL (sci_experiment 도구, 별도 저장소)
python -m sci_experiment.real_specs_extract --src <비공개폴더> --out real_specs_anonymized

# 2. JSONL → sentence_candidates.csv 변환 후 v2.8 검출
python scripts/run_detection.py --input <sentence_candidates.csv> --output results_d6
```
