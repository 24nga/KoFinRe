# KoFinRe 웹 평가기

> 단일 HTML — 서버 불필요, 데이터 서버 전송 없음 (100% 브라우저 처리)

## 사용법

### 옵션 1: 로컬에서 그대로 열기
1. `web/index.html` 더블클릭 (Chrome/Edge/Firefox 권장)
2. XLSX 파일 끌어놓기 또는 선택
3. 시트·컬럼 자동 식별 → 평가 시작
4. 결과 미리보기 확인 후 다운로드

### 옵션 2: GitHub Pages
저장소 Settings → Pages → Source: `main` 브랜치 `/web` 폴더 선택 → URL 발급

### 옵션 3: 정적 호스팅 (Netlify, Vercel, Cloudflare Pages)
`web/` 폴더 그대로 배포

## 기능

| 단계 | 동작 |
|---|---|
| 1️⃣ 파일 업로드 | XLSX 끌어놓기·클릭. 자동 시트·컬럼 인식 |
| 2️⃣ 설정 | 요구사항명·내용 컬럼·헤더 행 선택 |
| 3️⃣ 평가 | 10종 smell 자동 검출 (RegexDetector JS 포팅) |
| 4️⃣ 결과 | 요약 통계 + 200건 미리보기 + Excel 다운로드 |

## 평가 항목 (10종 smell)

| 코드 | 이름 | 검출 규칙 |
|---|---|---|
| S1 | 복합의무 | 의무 표현 2회 이상 |
| S2 | 불완전 | 응답·행위·대상 누락 |
| S3 | 모호어 | "적절히, 필요한, 실시간" 등 |
| S4 | 약한의무 | "~한다 / ~된다" 평서형 |
| S5 | 주체누락 | 수행 주체 미명시 |
| S6 | 정량부재 | 성능 키워드 + 숫자 없음 |
| S7 | 미정의약어 | 정의 없는 약어 |
| S8 | 범위모호 | "및/또는, 등, 포함" |
| S9 | 수동표현 | "되어야 한다" |
| S10 | 검증불가 | 정성 표현 + 측정 기준 부재 |

## 출력 Excel 구조

원본 시트 그대로 + 다음 컬럼 자동 추가:

| 추가 컬럼 | 내용 |
|---|---|
| `has_smell` | 1+ smell 검출 여부 (0/1) |
| `S1` ~ `S10` | 각 smell 검출 (0/1) |
| `검출_Smell_이름` | "S3(모호어), S5(주체누락)" 형태 |
| `수정_권고` | 자동 휴리스틱 교정 결과 (`[명세 필요: ...]` 마커 삽입) |
| `적용_규칙` | 어떤 교정 규칙 적용됐는지 |

추가 시트: `KoFinRe_요약` (전체 통계)

## 수정 권고 마커

휴리스틱 자동 교정으로 작성자가 채워야 할 부분을 명시:

| 원문 | 권고 |
|---|---|
| "실시간으로" | `[명세: 응답시간 임계값]` |
| "신속히" | `[명세: 시간 기준]` |
| "효율적으로" | `[명세: 측정 기준]` |
| "처리한다" | "처리해야 한다" (의무형 변환) |
| "권한관리 등을" | "권한관리 `[열거 명시 필요]`을" |

## 보안·프라이버시

- **모든 처리는 브라우저 내부** — 파일 업로드 0건
- 외부 서버·API 호출 0건 (SheetJS CDN만 — 라이브러리 로드)
- 영업 비밀 정의서·실기업 산출물 처리에도 안전

## 기술 스택

- **SheetJS** (xlsx@0.18.5) — XLSX 파싱·생성
- **순수 JavaScript** — 의존성 1개 (SheetJS)
- **RegexDetector** Python 포팅 — 10종 smell 정규식
- **휴리스틱 교정** Python 포팅 — S3·S4·S8 변환

## 한계 (서버형 대비)

- `kiwipiepy` 형태소 분석은 미지원 (브라우저 환경 제약)
- `MorphDetector` / `ChunkDetector` 미적용 — RegexDetector 단독
- 대용량 파일 (1만 건+) 시 브라우저 느려질 수 있음

본격 분석은 `kofinre` Python 패키지 사용 권장.
