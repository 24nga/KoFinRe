# KoFinRe v2.8 웹 평가기

> 단일 HTML — 서버 불필요, 데이터 서버 전송 없음 (100% 브라우저 처리)
> **19종 smell taxonomy** (Paska + IEEE 830 + ISO 29148 + INCOSE + EARS + CMMI REQM/RD + NCS 5 카테고리)

## 사용법

### 옵션 1: 로컬에서 그대로 열기
1. `web/index.html` 더블클릭 (Chrome/Edge/Firefox 권장)
2. XLSX 파일 끌어놓기 또는 선택
3. 시트·컬럼 자동 식별 → 평가 시작
4. 결과 미리보기 확인 후 다운로드

### 옵션 2: GitHub Pages
저장소 Settings → Pages → Source: `main` 브랜치 `/web` 폴더 선택 → URL 발급

### 옵션 3: 정적 호스팅 (Netlify, Vercel, Cloudflare Pages)
`web/` 폴더 그대로 배포

## 기능

| 단계 | 동작 |
|---|---|
| 1️⃣ 파일 업로드 | XLSX 끌어놓기·클릭. 자동 시트·컬럼 인식 |
| 2️⃣ 설정 | 요구사항명·내용 컬럼·헤더 행 선택 |
| 3️⃣ 평가 | **19종 smell** 자동 검출 (RegexDetector JS 포팅, v2.8) |
| 4️⃣ 결과 | 요약 통계 + 200건 미리보기 + Excel 다운로드 |

## 평가 항목 (19종 smell)

### 핵심 10종 (Paska + 한국 특화) — v1.0~v2.0

| 코드 | 이름 | 검출 규칙 |
|---|---|---|
| S1 | 복합의무 | 의무 표현 2회 이상 |
| S2 | 불완전 | 응답·행위·대상 누락 |
| S3 | 모호어 | "적절히, 필요한, 실시간" 등 |
| S4 | 약한의무 | "~한다 / ~된다" 평서형 |
| S5 | 주체누락 | 수행 주체 미명시 |
| S6 | 정량부재 | 성능 키워드 + 숫자 없음 |
| S7 | 미정의약어 | 정의 없는 약어 |
| S8 | 범위모호 | "및/또는, 등, 포함" |
| S9 | 수동표현 | "되어야 한다" |
| S10 | 검증불가 | 정성 표현 + 측정 기준 부재 |

### 확장 5종 (ISO 29148 / INCOSE / EARS) — v2.7

| 코드 | 이름 | 검출 규칙 |
|---|---|---|
| S11 | 구현편향 | Java/Oracle/Spring 등 + 의무 표현 |
| S12 | 부정문 | "~지 않아야 한다 / 안 된다 / 불가하여야" |
| S13 | 추측표현 | "가능하다면 / 추후 결정 / 필요시" |
| S14 | 수혜자불명 | 의무 표현 + persona(사용자·관리자 등) 부재 |
| S15 | 지시어모호 | "해당, 상기, 그것, 본 건" 등 |

### CMMI / NCS 4종 — v2.8 신규

| 코드 | 이름 | 검출 규칙 |
|---|---|---|
| **S16** | **필요성불명확** | "선호하는, 강제로, 임의로 정한, 독단적" |
| **S17** | **실현불가** | "100% 가용성, 99.9999%, 0초, 완벽한" 등 절대치 |
| **S18** | **추적ID부재** | 의무 표현 + ID 패턴 + 출처/근거 모두 부재 |
| **S19** | **제약카테고리불명** | "제약/제한" 시그널 + NCS 5 카테고리 0개 매칭 |

> NCS 5 카테고리: **TECH** (OS·플랫폼·프레임워크) / **BIZ** (예산·일정·한도) / **COMP** (개인정보·법규) / **OPS** (24시간·무중단) / **SEC** (MFA·암호화)

## 출력 Excel 구조

원본 시트 그대로 + 다음 컬럼 자동 추가:

| 추가 컬럼 | 내용 |
|---|---|
| `has_smell` | 1+ smell 검출 여부 (0/1) |
| `S1` ~ `S19` | 각 smell 검출 (0/1) — **19컬럼** |
| `검출_Smell_이름` | "S3(모호어), S5(주체누락)" 형태 |
| `수정_권고` | 자동 휴리스틱 교정 결과 (마커 삽입) |
| `적용_규칙` | 어떤 교정 규칙 적용됐는지 |

추가 시트: `KoFinRe_요약` (3 그룹별 통계 — 핵심 10 / 확장 5 / CMMI·NCS 4)

## 수정 권고 마커

휴리스틱 자동 교정으로 작성자가 채워야 할 부분을 명시:

| 원문 | 권고 |
|---|---|
| "실시간으로" | `[명세: 응답시간 임계값]` |
| "신속히" | `[명세: 시간 기준]` |
| "효율적으로" | `[명세: 측정 기준]` |
| "처리한다" | "처리해야 한다" (의무형 변환) |
| "권한관리 등을" | "권한관리 `[열거 명시 필요]`을" |
| "선호하는…" (S16) | `[정당화 필요]` 접두 |
| "100% 가용성" (S17) | `[현실적 수치 필요: SLO·SLA] 가용성` |
| "0초" (S17) | `[현실적 응답시간 필요]` |
| "완벽한" (S17) | `[측정 가능한 기준 필요]` |
| ID/출처 없는 의무문 (S18) | `[ID·출처 필요: REQ-XXX·근거 정책]` 접두 |
| "일부 제약" (S19) | `[카테고리 명시: TECH/BIZ/COMP/OPS/SEC] 제약` |

## 보안·프라이버시

- **모든 처리는 브라우저 내부** — 파일 업로드 0건
- 외부 서버·API 호출 0건 (SheetJS CDN만 — 라이브러리 로드)
- 영업 비밀 정의서·실기업 산출물 처리에도 안전

## 기술 스택

- **SheetJS** (xlsx@0.18.5) — XLSX 파싱·생성
- **순수 JavaScript** — 의존성 1개 (SheetJS)
- **RegexDetector v2.8** Python 포팅 — 19종 smell 정규식 + NCS 5 카테고리 사전
- **휴리스틱 교정** Python 포팅 — S3·S4·S8 변환 + **S16·S17·S18·S19 신규 마커 4종**

## 한계 (서버형 대비)

- `kiwipiepy` 형태소 분석은 미지원 (브라우저 환경 제약)
- `MorphDetector` / `ChunkDetector` 미적용 — RegexDetector 단독
- 의미 임베딩 기반 S20 Consistency 미적용 (서버형 v2.9에서 제공 예정)
- 대용량 파일 (1만 건+) 시 브라우저 느려질 수 있음

본격 분석은 `kofinre` Python 패키지 사용 권장.
